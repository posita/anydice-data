function: test ROLL:s
{
  \
  First find the highest value of any die
  \
  HIGH: 1@ROLL

  \
  Next return the highest value, plus one for each extra die that has the same value.
  We do this by subtracting one from the count of that high result - because we don't get the
  bonus from the first result of that value, only second and subsequent.
  \
  result: HIGH + ([count HIGH in ROLL]-1)
}

ROLL: 3d10

output [test ROLL]

\
For 3d10, the lowest possible roll is 1, 1, 1; by the rules above this should result in 3 (1 + 2 extra 1s).
So why is there any chance at all of getting a 2?
\

ROLL: 5d10

output [test ROLL]

\
For 5d10, the lowest possible roll is 1, 1, 1, 1, 1 = 5... and yet there's still apparently a chance of
the result being 2, 3, up to a 0.5% chance of rolling a 4?
\
