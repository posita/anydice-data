output(4 + 1d{2..11}) - (1d6 + 1d{2..7}) + 1 >= 3 named "Damage calc"
\ 1st brackets is the attacker. Input initial attack roll result and ".." bonus range. \
\ 2nd brackets is the defender. Input ".." bonus range. It is not possible to know initial defend roll result beforehand. \
\ Use "+ 0" before the >= if there is no +1 damage modifier i.e HaiQ Fate Echo \
\ After the >= is input defender's current HP \
\ If output 1(true) is over 50% then theoretically the defender has higher chance to survive by evading instead. \

output 1d6 >= 4 named "Evade"
\ Change the number after >= to the attacker's initial attack roll result. If it is 6 then change >= into only = \
\ Outputs #0 is false/fail to meet goal chance. Compare evasion's "#1" with damage calc's "#0" as defender\
\ If both defend/evade is 50/50 it is better to retain a current HP of 2 or higher rather than survive with 1 HP. So evade. \
