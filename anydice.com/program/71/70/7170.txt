  OVERVIEW:
  The following is a breakdown of the skill checks system for my RPG.

  In certain situations, players must make skill checks to see how their characters fair. In addition, non-player characters
  may need to make similar checks; in this case, the GM makes them. These checks are somewhat random, but rely heavily on the
  skills that a specific character has.

  There are two different types of checks, either unopposed, or opposed. In either case, the player rolls d10s (10-sided dice).
  The total number of d10’s that the player rolls is equivalent to the number of skill points they have in the skill they are
  checking. For example, if a character has a Strength skill of 3 and is attempting to make a Strength check, the player or GM
  that represents them, rolls three d10s.

  The player or GM is also always trying to beat a particular value with their roll, which is dependent on the difficulty and
  type of check they are trying to make (described below). For each roll that beats that specified value, the total value of the
  roll is increased by one. For example, if the player is trying to beat a 4 and, in rolling three d10’s, two of them were higher than
  4, the result of the roll is a 2.

  The following are simulations for this system, which output correct data as to the probabilities of possible outcomes.

  Both uncontested and contested checks are represented.
\

\ -------- TOTAL NUMBER OF LEVELS THAT WILL BE SIMULATED ------- \

                           STATLVLS: 20

\ -------------------------------------------------------------- \

\ UNCONTESTED NOTES:

  This is the breakdown of what must be met in order to succeed in a check:

  Easy          : Must beat a 3 (i.e. 4 or higher) at least 1 time
  Simple        : Must beat a 4 (i.e. 5 or higher) at least 2 times
  Tough         : Must beat a 5 (i.e. 6 or higher) at least 3 times
  Challenging   : Must beat a 6 (i.e. 7 or higher) at least 4 times
  Inconceivable : Must beat a 7 (i.e. 8 or higher) at least 5 times

  The following are representations of possible stat checks against various difficulties. These
  representations simulate the rules for rolling uncontested skill checks. For example:

  In the Easy check, there are three 0's (failures), and seven 1's (successes) simulating the
  probability of rolling higher than 3. If the total value is at least 1, it means we rolled at
  least one success and the result of the check is 1 (success).

\

\--------------------- CODE STARTS HERE ------------------------- \

loop STAT over {1..STATLVLS}
{
    output STATd{0:3,1:7} >= 1 named "Easy with stat of [STAT]"
    output STATd{0:4,1:6} >= 2 named "Simple with stat of [STAT]"
    output STATd{0:5,1:5} >= 3 named "Tough with stat of [STAT]"
    output STATd{0:6,1:4} >= 4 named "Challenging with stat of [STAT]"
    output STATd{0:7,1:3} >= 5 named "Impossible with stat of [STAT]"
}

\---------------------- CODE ENDS HERE --------------------------- \

\ CONTESTED NOTES: 

  If a character is attempting to take an action that relies on their skill, but which can also be affected by the skill
  of another creature or object, they roll an Opposed Check. To be successful in an Opposed Check, a character must usually
  beat a 5 at least as many times as the character that they are opposed with. For example, if a character finds themself in
  opposition with a Savage and is required to make a Discipline check, each rolls as if they were making a Tough
  Unopposed Check. However, the result of the check is instead determined by whoever rolls more successes (values higher than 5).
  In the case of a tie, the defending/opposing character fails and the character who first attempted the action is successful.

  ADVANTAGE AND DISADVANTAGE:
  Certain situations may dictate that a certain character has advantage or disadvantage. Characters with advantage gain -1 to the
  value they must beat or +1 if  disadvantaged. They can have up to -2 or +2. If both characters have disadvantage, or if they both
  have advantage, subtract the value closer to zero from both. For example, if a character has -2 and the other has -1, subtracting
  the value closer to zero, -1, from both yields -1 and 0 in terms of the advantages. If both values are equal, subtract that value
  from both, e.g. +1 and +1 becomes +0 and +0.

  DEALING DAMAGE:
  The amount of damage that is evaluated is determined by the result of this roll as well. This is done by subtracting the number of successes that
  the opposition rolled from the number of successes that the character who attempted the attack rolled. For example, if a character were to attack
  another, each would roll as if it were a normal Opposed Check (described above). If the attacking character rolled a total of 4 successes and the
  defending character rolled only 2, the attacking character would deal 2 points of damage. If the defending character rolls more, the attack simply
  misses.

\

\--------------------- CODE STARTS HERE ------------------------- \

loop STAT over {1..STATLVLS}
{
\   You can set the advantage of the player or enemy here:  \
    ADVANTAGE: 0
    ENEMYADV: 0

    STATSEQ: {0:5-ADVANTAGE,1:5+ADVANTAGE}
    ENEMYSTATSEQ: {0:5-ENEMYADV,1:5+ENEMYADV}

    loop ENEMY over {1..STATLVLS}
    {
        output STATdSTATSEQ >= ENEMYdENEMYSTATSEQ named "Winning an offensive check against a LVL [ENEMY] (+[ENEMYADV] ATG) with stat LVL [STAT] (+[ADVANTAGE] ATG)"
        output STATdSTATSEQ - ENEMYdENEMYSTATSEQ named "Damage dealt against enemy with defense of [ENEMY] (+[ENEMYADV] ATG) with offense of [STAT] (+[ADVANTAGE] ATG)"
    }
}

\---------------------- CODE ENDS HERE --------------------------- \
