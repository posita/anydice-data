set "maximum function depth" to 5
\Limits the depth of recursive functions to 5. The only recursive function in here is the explosive roll for old editions of oWoD that make you reroll all “10”s for possible additional successes.\
\The odds of having to roll the same die 5 times are actually one in hundred thousand, and odds of having to roll 6 times are one in a million\

\Here is the list of functions used in this program or supplements:\

function: diceroll DIE:n vs DIFFICULTY:n {
\if DIE = 10 {result: 2}\
\Uncomment the line above if rolling for a trait in which the character is specialized or 10 grants two successes for any other reason.\
\Alternatively, use the function provided below.\
if DIE >= DIFFICULTY {result: 1}

result: 0
}

\For technical reasons, die is not specified yet. But it only works with D10 if you use specialty roll.\
\Result of 1 means a success, -1 means rolling a “1” and losing a success, 0 means neither (a fail).\

function: specialtyroll DIE:n vs DIFFICULTY:n {
if DIE = 10 {result: 2}

function: hatsevaluate ROLL:s {
\accept a sequence of numbers. 1s subtract from final number. \
if (ROLL < 0) > (ROLL >0) {
\if less than 0 successes, we botch automatically\
result: -1000
}
if (ROLL > 0) > (ROLL < 0) {
\if more successes than botches, we can return the result\
result: ROLL
}
result: 0
\if neither more or less, the result is 0\
}

function: smartevaluate ROLL:s {
\It accepts a sequence of numbers that are roll results from the function(s) above: 1s being success dice, -1 being botch dice, 0 being neither, and 2s indicating rolling a 10 for a roll within specialty.\
if (ROLL > 0) > 0 {
\First, in the line above, it checks if there are successes (results of >0, namely 1 or 2) in a given sequence.\
result: ROLL
\If there are any, it returns the sequence as a number, which means summing all of it’s numbers, returned as a roll result. As we give the result, function is stopped.\
\Positive values of this sum are different amounts of successes, 0 and negative are Fails.\
}
if (ROLL = -1) > 0 {
\If no successes are found, it checks for “1”s, or botch dice.\
result: -1000
\If there are any, as we already know that there are no successes, it declares a Botch (results in -1000)\
\Because of possible result of -1000, this is not usable for calculating mean.\
}
result: 0
\If neither “1”s nor successes are rolled, it returns 0 as the only possible result.\
}

\Rules for applying automatic successes to botched rolls or losing them due to 1s are not specified in the core book.\
\If you have an automatic success granted by spending Willpower, you can just use this program as it is, counting all the botches and fails as marginal successes (one success acquired), and adding one success to any success results. \

function: nobotchevaluate ROLL:n {
if (ROLL < 0) {result: 0}
result: ROLL
}

\You can’t botch in some combat situations, like a damage roll. Or a botch can simply not bother you in some cases.\
\So the function above can’t return less than 0, this result covers all botches and all fails.\
\Other results are degrees of success.\
\The main usage of this function is to calculate average amount of successes per roll with given dicepool against a given difficulty.\

loop DIFFICULTY over {5..10} {
loop DICEPOOL over {1..8} {
DICEROLL: [diceroll d10 vs DIFFICULTY]
\output [smartevaluate DICEPOOL d DICEROLL] named "Dicepool of [DICEPOOL] vs difficulty of [DIFFICULTY]"}\
output [hatsevaluate DICEPOOL d DICEROLL] named "Dicepool of [DICEPOOL] vs difficulty of [DIFFICULTY]"}

output 0 named "_______________________________________"
\Just a gap, so outputs invoked for different combinations are more distinct\
}

\Specify the ranges of the DICEPOOL (between 1 and ~250) and the DIFFICULTY (between 2 and 10)\
\Every dicepool within range will be rolled against every difficulty within range. \
\The loops roll a given amount of dice against a given difficulty, evaluating the sequences for a possible Botches, then return the result of a roll.\
\If -1000 is returned, it is a Botch. Positive numbers are different degrees of success, other negative are different degrees of fail.\

\What can you do with AnyDice.com:\
\”Normal” gives you the odds of every possible result. “At least” and “At most” give the odds of getting at least or at most this amount of points respectively, useful when, for example, you need at least a given amount of successes. “Summary” contains some useful statistical things like calculated mean and standard deviation, but check if it is representative.\
