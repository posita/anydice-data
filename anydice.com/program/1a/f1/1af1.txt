\Function for the main dice roll. Returns 21 if a straight and 0 if a triple. It also stores the true dice value in any of those cases, to be accessed later.\
\A type of 0 is an attack die. A type of any other value is an evasion die.\
function: lunardice A:n and B:n and C:n type D:n {
  \Triples\
  if A = B {
    if B = C {
      if D = 0 {
        AROLLVALUE: A + B + C
      }
      else {
        EROLLVALUE: A + B + C
      }
      result: 0
    }
  }
  \Straights\
  if (A != B) & (A != C) & (B != C) {
    if (B = A + 2) & (B = C + 1) {
      if D = 0 {
        AROLLVALUE: A + B + C
      }
      else {
        EROLLVALUE: A + B + C
      }
      result: 21
    }
    if (B = A + 1) & ((B = C - 1) | (B = C + 2)) {
      if D = 0 {
        AROLLVALUE: A + B + C
      }
      else {
        EROLLVALUE: A + B + C
      }
      result: 21
    }
    if (B = A - 1) & ((B = C - 2) | (B = C + 1)) {
      if D = 0 {
        AROLLVALUE: A + B + C
      }
      else {
        EROLLVALUE: A + B + C
      }
      result: 21
    }
    if (B = A - 2) & (B = C - 1) {
      if D = 0 {
        AROLLVALUE: A + B + C
      }
      else {
        EROLLVALUE: A + B + C
      }
      result: 21
    }
  }
  \Standard roll\
  result: A+B+C
}

\Doing the damage calculator shouldn't be too difficult. A bunch of variables to store attack and evade, or maybe just one for the difference...which is then checked against damage range values, multiplied by the Damage Step, and added or subtracted from the Base Damage. Values higher than Max Damage are returned as Max Damage, values below the lower Damage Range bound are returned as 0, special automiss states are returned as negatives (except when calcing averages), autohits are returned as Max Damage, and crits are returned as double Max Damage. This will return fairly precise values on all relevant stats, up to and including a true miss rate.\

\Using this to handle pure KE or TE damage would be easy enough. Modifying it for missiles wouldn't be hard either. Not sure how to handle mixed damage types, ion weapons, or Bonus Damage though. (The latter could probably be handled as a boolean modifier.) Melee might be tricky, as you'd want to determine return damage in the case of counters.\

\This also doesn't exactly determine the average difference between attack and evade, where autohits are not in effect. While AnyDice calcs a mean for you, the way special rolls are returned renders it inaccurate. If only I could return non-numerical values...I could probably fudge it by forcing it to only return when there are no special states on the dice, but I'm not even sure how to force it to NOT return a value. I can force it to return based on other subsets no problem, but forcing a standard-only return might be tricky.\

\Function Syntax: damage <Attack roll> <Attack bonus> <Evasion roll> <Evasion bonus> <base damage> <damage step> <max damage>\
\AROLL and EROLL are lunardice calls. PRECISE is a boolean. All others are integers.\
function: damage AROLL:n ABONUS:n EROLL:n EBONUS:n BDAM:n DSTEP:n MDAM:n PRECISE:n {
  \Automiss and Autoevade\
  if AROLL = 0 {
    result: 0
  }
  if (AROLL != 21) & (EROLL = 21) {
    result: 0
  }
  \Autohit and Critical\
  if (AROLL = 21) & (EROLL != 21) {
    \Autohit when attack would otherwise miss\
    if ((AROLL - EROLL) * DSTEP) + BDAM <= 0 {
      result: MDAM
    }
    \Crit when attack would hit\
    else {
      if PRECISE != 0 {
        result: MDAM*3
      }
      else {
        result: MDAM*2
      }
    }
  }
  \Dual Straights\
  if (AROLL = 21) & (EROLL = 21) {
    AROLL: AROLLVALUE
    EROLL: EROLLVALUE
  }
  \Normal Rolls - Equal Attack and Evasion\
  if AROLL = EROLL {
    result: BDAM
  }
  \Normal Rolls - Attack greater than Evasion\
  if AROLL > EROLL {
    \Don't surpass Max Damage!\
    if ((AROLL - EROLL) * DSTEP) + BDAM >= MDAM {
      result: MDAM
    }
    result: (AROLL - EROLL) * DSTEP + BDAM
  }
  \Normal Rolls - Evasion greater than Attack\
  if EROLL > AROLL {
    \Don't deal negative damage!\
    if ((AROLL - EROLL) * DSTEP) + BDAM <= 0 {
      result: 0
    }
    result: (AROLL - EROLL) * DSTEP + BDAM
  }
}
