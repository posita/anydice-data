\Function for the main dice roll. Returns its value +100 if a straight and its value negative and minus 100 if a triple.\
\A type of 0 is an attack die. A type of any other value is an evasion die.\
\Because a straight relies on the independent values of each die, and because I couldn't figure out how to do it other way, each die is called individually.\
function: lunardice A:n and B:n and C:n {
  \Triples\
  if A = B {
    if B = C {
      result: (0-100) - (A + B + C)
    }
  }
  \Straights - probably a better way to do this rather than brute forcing it?\
  if (A != B) & (A != C) & (B != C) {
    if (B = A + 2) & (B = C + 1) {
      result: A + B + C + 100
    }
    if (B = A + 1) & ((B = C - 1) | (B = C + 2)) {
      result: A + B + C + 100
    }
    if (B = A - 1) & ((B = C - 2) | (B = C + 1)) {
      result: A + B + C + 100
    }
    if (B = A - 2) & (B = C - 1) {
      result: A + B + C + 100
    }
  }
  \Standard roll\
  result: A+B+C
}

\Function Syntax: damage <Attack roll> <Attack bonus> <Evasion roll> <Evasion bonus> <base damage> <damage step> <max damage> <minimum of damage range> <maximum of damage range> <Precise boolean>\
\AROLL and EROLL are lunardice calls. PRECISE is a boolean. All others are integers.\
\This logic may not be an exact implementation - I forget exactly how triples on evasion work at this hour and melee has its own issues - but it's more than accurate enough for my purposes currently.\
function: damage AROLL:n ABONUS:n EROLL:n EBONUS:n BDAM:n DSTEP:n MDAM:n MINSTEP:n MAXSTEP:n PRECISE:n {
  \Automiss and Autoevade\
  if AROLL < -100 {
    result: 0
  }
  if (AROLL < 100) & (EROLL > 100) {
    result: 0
  }
  \Autohit and Critical\
  if (AROLL > 100) & (EROLL < 100) {
    \Autohit when attack would otherwise miss\
    if (((AROLL - 100) + ABONUS) - (EROLL + EBONUS)) < MINSTEP {
      result: MDAM
    }
    \Crit when attack would hit\
    else {
      if PRECISE != 0 {
        result: MDAM*3
      }
      else {
        result: MDAM*2
      }
    }
  }
  \Dual Straights - should be redundant, but here just in case.\
  if (AROLL > 100) & (EROLL > 100) {
    AROLL: AROLL - 100
    EROLL: EROLL - 100
  }
  \Evade Botch\
  if (AROLL > -100) & (EROLL < -100) {
    result: MDAM
  }
  \Normal Rolls - Equal Attack and Evasion\
  if (AROLL + ABONUS) = (EROLL + EBONUS) {
    result: BDAM
  }
  \Normal Rolls - Attack greater than Evasion\
  if (AROLL + ABONUS) > (EROLL + EBONUS) {
    \Don't surpass Max Damage!\
    if ((AROLL + ABONUS) - (EROLL + EBONUS)) >= MAXSTEP {
      result: MDAM
    }
  }
  \Normal Rolls - Evasion greater than Attack\
  if (EROLL + EBONUS) > (AROLL + ABONUS) {
    \Don't deal negative damage!\
    if ((AROLL + ABONUS) - (EROLL + EBONUS)) < MINSTEP {
      result: 0
    }
  }
result: ((AROLL + ABONUS) - (EROLL + EBONUS)) * DSTEP + BDAM
}

\Standard evasion for Heavyweights is +3, standard evasion for Lightweights is +10. At least as it stands currently. That may well change if HP values get too wildly far appart or I can't control PC APU frame stats effectively. Under my current calculations, with health counts as they are and attack at +7, all three weight classes have a reasonably equivalent time-to-kill with the weapons I've tested.\
\The way part builds cross over involves selection of parts currently - and a bonus given when all parts are of the same weight class. I might reduce the Heavy and Light health bonuses to 10% to keep the crossover advantage from being too large.\
