output 1@3d6 named "Highest of 3d6"
\ This notation means: "Take the SEQUENCE of the die roll after '@', and then use only the X highest result." In this case, roll 3d6 and discard the two lowest dice \
output {1,2}@3d6 named "Discard lowest of 3d6"
\ You can use this to add together things from the result sequence. Basically, roll 3d6, keep and add together the highest and second highest dice \

output 5d(1d6 >= 5) named "5d6 dicepool, TN 5"
\ And here's the shortest way to notate a dicepool. The "d" notation in anydice is rather... Flexible. Here, we use the result of an operation as a probability curve itself.

A die can be said to be a probability curve in itself. A "d6" is a linear curve, just the probability of each result is equal. But you can have dice that have the same result on many face: you could have a die {1, 1, 2, 3, 4, 5}, right? The result 1 has a 2/6 chance of being rolled. That's a slightly wonky curve.

So nothing stops you from rolling even wonkier curves, like (1d6 >= 5). It's actually a pretty SIMPLE curve, because (1d6 >= 5) evaluates as "when is 1d6 equal or greater than 5?", and it outputs just 0 and 1. If you wrote it down as a die, you get {0, 0, 0, 0, 1, 1}.

That die can be shortened to {0, 0, 1} of course, and in anydice notation you can do {0:2, 1}, meaning "two zeroes". You've already seen {1..3}, which is the same as {1, 2, 3}.

Mathematically, {0,0,0,0,1,1} is not distinct from "0:66%, 1:33%", which is what you get out of the (1d6 >= 5). Because of this, you can plug an arbitrary curve in there and use it a wonky die.
