function: diceroll DIE:n vs DIFFICULTY:n {
\if DIE = 10 {result: 2}\
\Uncomment the line above if rolling for a trait in which the character is specialized or 10 grants two successes for any other reason.\
if DIE >= DIFFICULTY {result: 1}
if DIE = 1 {result: -1}
result: 0
}
\This sets the rule for rolling a die in oWoD.\
\For technical reasons die is not specified yet. But it only works with D10.\
\1 means a success, -1 means rolling a “1” and losing a success, 0 means neither.\

function: evaluate ROLL:s {
if (ROLL > 0) > 0 {
result: [highest of 0 and ROLL]
}
result: [highest of -1 and ROLL]
}
\This function accepts a sequence of numbers that are roll results from the function above.\
\It checks if there are successes (results of 1) in a given sequence.\
\If there are any, it returns either 0 (a fail, if all the successe are canceled) or a greater number (the amount of successes). It can’t be a botch, as at least one success was rolled.\
\If no successes are spotted, it is either a botch or a fail. The function returns -1 (a botch) if at least one such result was rolled and 0 (a fail) if neither successes nor “1”s were rolled.\

\Sets the die type and difficulty.\
AUTOSUCCESSES: 0

\Not fully supported yet as rules for using autosuccesses with fails and botches aren’t fully understood yet.\
